# Tutorial : StatefulSet MySQL avec persistance sur Kubernetes

## Prérequis

- Cluster Kubernetes fonctionnel (`minikube`, `kind`, ou cloud)
- `kubectl` configuré

---

## 1. Créer le Secret

```yaml
# secret.yaml
apiVersion: v1
kind: Secret
metadata:
  name: mysql-secret
type: Opaque
stringData:
  root-password: "MonMotDePasse123!"
```

```bash
kubectl apply -f secret.yaml
```

---

## 2. Créer les Services

```yaml
# services.yaml
apiVersion: v1
kind: Service
metadata:
  name: mysql        # Headless — DNS interne stable
spec:
  clusterIP: None
  selector:
    app: mysql
  ports:
    - port: 3306
---
apiVersion: v1
kind: Service
metadata:
  name: mysql-nodeport   # Accès externe
spec:
  type: NodePort
  selector:
    app: mysql
  ports:
    - port: 3306
      targetPort: 3306
      nodePort: 30306
```

```bash
kubectl apply -f services.yaml
```

---

## 3. Créer le StatefulSet

```yaml
# statefulset.yaml
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: mysql
spec:
  serviceName: mysql
  replicas: 1
  selector:
    matchLabels:
      app: mysql
  template:
    metadata:
      labels:
        app: mysql
    spec:
      containers:
        - name: mysql
          image: mysql:8.0
          ports:
            - containerPort: 3306
          env:
            - name: MYSQL_ROOT_PASSWORD
              valueFrom:
                secretKeyRef:
                  name: mysql-secret
                  key: root-password
            - name: MYSQL_DATABASE
              value: mydb
          volumeMounts:
            - name: mysql-data
              mountPath: /var/lib/mysql
          resources:
            requests:
              memory: "512Mi"
              cpu: "250m"
            limits:
              memory: "1Gi"
              cpu: "500m"
          livenessProbe:
            exec:
              command: ["mysqladmin", "ping", "-h", "localhost"]
            initialDelaySeconds: 30
            periodSeconds: 10
          readinessProbe:
            exec:
              command: ["mysql", "-h", "localhost", "-e", "SELECT 1"]
            initialDelaySeconds: 15
            periodSeconds: 5
  volumeClaimTemplates:
    - metadata:
        name: mysql-data
      spec:
        accessModes: ["ReadWriteOnce"]
        storageClassName: standard
        resources:
          requests:
            storage: 10Gi
```

```bash
kubectl apply -f statefulset.yaml
```

---

## 4. Vérifier le déploiement

```bash
# État du StatefulSet
kubectl get statefulset mysql

# Pod créé
kubectl get pod mysql-0

# PVC créé automatiquement
kubectl get pvc mysql-data-mysql-0

# Logs du pod
kubectl logs mysql-0
```

Résultat attendu :

```
NAME    READY   AGE
mysql   1/1     30s
```

---

## 5. Tester la connexion

**Depuis l'intérieur du cluster :**

```bash
kubectl exec -it mysql-0 -- mysql -u root -p mydb
```

**Depuis l'extérieur (NodePort) :**

```bash
mysql -h <IP-du-nœud> -P 30306 -u root -p mydb
```

> Avec minikube : `minikube ip` pour obtenir l'IP du nœud.

---

## 6. Vérifier la persistance

```bash
# Supprimer le pod (il sera recréé automatiquement)
kubectl delete pod mysql-0

# Le PVC et les données sont conservés
kubectl get pvc mysql-data-mysql-0
```

---

## Récapitulatif des fichiers

```
.
├── secret.yaml
├── services.yaml
└── statefulset.yaml
```

```bash
# Tout déployer d'un coup
kubectl apply -f secret.yaml -f services.yaml -f statefulset.yaml

# Tout supprimer (le PVC persiste volontairement)
kubectl delete -f statefulset.yaml -f services.yaml -f secret.yaml
```

> ⚠️ Pour supprimer aussi les données : `kubectl delete pvc mysql-data-mysql-0`
